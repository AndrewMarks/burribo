**Deployment approach:**
GitHub Pages can build Jekyll directly from the repository. The `github-pages` gem is already in Gemfile for compatibility.

Two deployment options:
1. Push to main branch → GitHub builds automatically
2. Use gh-pages branch for built site

Recommend Option 1 (simpler) since we're using standard Jekyll.

Ensure GitHub Pages configuration is correct:

3. Check Gemfile includes github-pages gem:

<task type="auto">
  <name>Task 5: Push to GitHub for deployment</name>
  <files></files>
  <action>
Push changes to GitHub to trigger deployment:

1. Verify remote is configured:
   ```bash
   git remote -v
   ```
   Should show origin pointing to GitHub repo.

2. Push to main branch:
   ```bash
   git push origin master:main
   ```
   (Or just `git push` if remote tracking is set up)

3. GitHub Actions will build and deploy automatically.

Note: If no remote is configured yet, user will need to:
   - Create repo on GitHub named "burribo"
   - Add remote: git remote add origin https://github.com/andrewmarks/burribo.git
   - Push: git push -u origin main

After push, the site will be available at:
https://andrewmarks.github.io/burribo/
  </action>
  <verify>Git push completes successfully</verify>
  <done>Changes pushed to GitHub, deployment triggered</done>
